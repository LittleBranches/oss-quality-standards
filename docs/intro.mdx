# OSS Quality Standards

A **model-agnostic, central quality standards system** for all repositories under the
[LittleBranches](https://github.com/LittleBranches) GitHub organisation.

## Purpose

Every AI model (Claude, Copilot, Grok, Gemini, …) that works on a LittleBranches
repository should follow the same reviewer instructions. Instead of bootstrapping those
instructions from scratch every session, load this system once and enforce everything:

- PR review checklist
- Branch hygiene rules
- Quality gate steps
- Component structure convention
- Definition of Done
- AI Collaboration & Permission Protocol

## How to load (universal trigger)

Paste this into any AI chat session:

```
Load reviewer instructions from AGENTS.md in LittleBranches/oss-quality-standards
```

The model fetches the `AGENTS.md` barrel file at the root of this repo and loads all rules.

For **sensitive / internal rules** (banned content, private refs, competitor references),
also load the private barrel:

```
Load reviewer instructions from AGENTS.md in LittleBranches/oss-quality-standards-private
```

## Scope

**Current:** React projects using MUI (Material UI).

Sections covering component structure, API contract, naming, accessibility, testing, and documentation strategy are React + MUI specific. Sections covering AI collaboration, branch hygiene, quality gate, PR review workflow, and Definition of Done are framework-agnostic and apply to any LittleBranches repository.

**Planned:** Vue and Angular equivalents will follow the same barrel structure when added.

## Repository structure

```

├── AGENTS.md                  ← the single barrel file — load this
├── docs/                      ← full documentation (this site)
│   ├── intro.mdx

│   ├── pr-review-workflow.md
│   ├── definition-of-done.md
│   ├── component-structure.md
│   └── quality-gate.md
├── scripts/
│   ├── quality-gate.js        ← reusable quality gate runner
│   └── check-structure.js     ← component folder structure checker
└── .github/
    └── pull_request_template.md
```

## Architecture

| Repo | Purpose |
|---|---|
| `LittleBranches/oss-quality-standards` | Public — career/portfolio showcase, all non-sensitive standards |
| `LittleBranches/oss-quality-standards-private` | Private — extends public with banned content, internal refs |

## Adding this to a new repository

Create a pointer file at the root of the new repo:

```markdown
# AI Reviewer Instructions

Load: https://raw.githubusercontent.com/LittleBranches/oss-quality-standards/main/AGENTS.md
```

Or simply tell the AI assistant at the start of each session to load the trigger phrase above.
